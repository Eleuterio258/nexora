## Consistency notes

- **Placeholder icons to replace**: most screens use stock Android icons (`ic_menu_camera`, `stat_sys_data_bluetooth` for NFC, `ic_launcher_foreground` for fingerprint) — when generating the designs, create a coherent custom icon set for the 7 check-in methods (Manual, QR, Facial, Selfie+GPS, PIN, NFC, Fingerprint) and reuse them consistently across `funcionario_home` and each method's individual screen. *(Done in code: see `ic_method_*.xml` and `ic_nav_*.xml` under `res/drawable/`.)*
- **Repeated pattern**: the 7 check-in screens (facial, fingerprint, selfie+GPS, PIN, QR, NFC, manual) follow the same skeleton — title, instruction, visual/icon area, Clock In/Clock Out toggle, action button, optional progress indicator. Generating them as variations of a single template saves work.
- **"Empty" list screens**: Team, Devices, Alerts, Occurrences, Reports, Vacation Requests only have a title + RecyclerView in the XML — the list-item design is left to the AI's discretion; use the flat-row pattern (no border/shadow) with a hairline divider and status badge as the common baseline.
- **Persistent chrome**: screens 3, 12, 17, 23, 25 (Home, History, Chat, Requests, Profile) are reached via the bottom nav (#36) and should be mocked up together with it visible, since it's now a permanent fixture of the employee experience — not a standalone screen.
- **Two bottom nav bars, one icon set**: the Gestor bar (#37) does not get its own icon designs — it reuses the same 5 drawables from the Colaborador bar (#36) under different labels (Dashboard→Home icon, Team→Profile icon, Vacations→Vacation icon, Reports→Modules icon, More→More icon). Keep any regenerated icon set shared between both prompts, not designed independently. Also note managers land on the Colaborador Home bar by default; the Gestor bar is reached by explicit navigation, not automatically after login.
